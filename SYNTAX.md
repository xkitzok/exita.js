## Syntax Reference

### Module Declaration

Every `.exj` file that exports components begins with `Add.Module` pointing to its header file. for example

```
Add.Module [ComponentName.hxj]
```

This replaces `export` statements. The compiler generates the header and makes all top-level components available to importers.

### Importing Modules

Consume a module by referencing its header.

```
Add.Module [Button.hxj]
Add.Module [./path/to/Header.hxj]
Add.Module [~ui/Card.hxj]
```

No `import` keyword. No relative path confusion. The header contains the type contract.

### Signals

Inside a component, `let` creates a reactive signal automatically.

```
let count = 0
let name = "Exita"
let items = []
```

Read the value with `.value` (added by the compiler):

```
<h1>{count.value}</h1>
```

Assign to update and trigger re-render:

```
count.value++
name.value = "New Name"
```

The compiler handles the `.value` insertion. You write `let count = 0` and use `count` normally.

### Components

Components are functions that return JSX. Top-level components are auto-exported when `Add.Module` is present.

```
function Greeting({ name = "World" }) {
  return <h1>Hello {name}</h1>
}
```

Props with default values infer their types automatically.

### Async Components

Components can be `async`. The runtime handles loading states.

```
async function DataView() {
  const data = await fetch('/api/data').then(r => r.json())
  return <pre>{JSON.stringify(data, null, 2)}</pre>
}
```

### Built-in Hooks

These are globally available. No imports required.

```
const videoRef = ref()
onMount(() => videoRef.current.play())
```

`ref()` creates a DOM reference. `onMount()` runs code after the component mounts. `onCleanup()` runs on unmount.

### Styling

Styles defined in a `<style>` block are automatically scoped and extracted.

```
function Card() {
  return (
    <>
      <style>{`
        .card { border: 1px solid #ccc; padding: 16px; }
        .card:hover { box-shadow: 0 2px 8px rgba(0,0,0,0.1); }
      `}</style>
      <div class="card">Content</div>
    </>
  )
}
```

At build time, styles are extracted to `.exj.css` files. Unused styles are eliminated.

### Headers

`.hxj` files are the public contract for a module.

```
Add.Module [Button.exj]

export interface ButtonProps {
  variant?: "primary" | "secondary"
  size?: "sm" | "md" | "lg"
  children: any
}

export const Button: (props: ButtonProps) => JSXElement
export const __signals: ["isHovering"]
```

Headers are auto-generated by the compiler. You can write them manually for library wrappers or API contracts before implementation.
